    • 1.2 The FluidTYPO3 extension familiy
      short overview about which extension is needed for what.
      something like:
      • EXT:flux
        Flux is a replacement API for TYPO3 FlexForms - with interfaces for Fluid, PHP and TypoScript - Flux is a dependency for:
        • EXT:fluidpages
        • EXT:fluidcontent
        • fluidcontent_core
      • EXT:fluidpages
        FluidPages enables page template selection and rendering รก la TemplaVoila - but using Fluid templates. It uses Flux to enable highly dynamic configuration of variables used when rendering the template.
      • EXT:fluidcontent
        FluidContent lets you write custom content elements based on Fluid templates. Each content element and its possible settings are contained in a single Fluid template file. Whole sets of files can be registered and placed in its own tab in the new content element wizard, letting you group your content elements.
      • EXT:fluidcontent_core FluidContentCore is a replacement for CSS_STYLED_CONTENT
      • EXT:vhs
        VHS is a libray extensions which provides assistance tools like viewhelpers and special classes for Extbase/Fluid. A collection of ViewHelpers to perform rendering tasks which are not natively supported by Fluid
      • EXT:builder
        TYPO3 cli-script tool for creating FluidTYPO3 based extension
      • EXT:site
        SiteKickstarter is a fire-and-forget install helper which when installed will install every standard FluidTYPO3 dependency - and when "deployed", creates the pages, mount points, TypoScript and configuration you need to run a FluidTYPO3-based site.
